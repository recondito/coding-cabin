/code/ - Coding Cabin

Please take a look at /dev/ - Developer's Diaries to see the project this blog is based on. The sole purpose of the Coding Cabin is to serve as an improved successor of Developer's Diaries. This time using a much better template while keeping the content the same. Also using Sanity instead of Decap CMS.

This blog was created during my third week of my ATDEV's Internship Program. So far I've been having the time of my life, and this website should serve to document my progress through my time as an intern. I have written more code in three weeks than I've written in the entire year, probably because I'm much more motivated now that I feel like I have a purpose.

/dev/ - Developer's Diaries was inspired in 4chan's minimalistic and almost "old" web design, along with the naming convention. I'm not a big 4chan user but I'm not gonna lie, the website is really well designed which is what motivated me to base my first iteration of the blog on it. However in a more professional environment, that layout can come off as a little bit "lazy". So this time I chose to use a better looking template.

About me

I have been interested in Software Development since the age of 7 but didn't really have the guts to actually code until I was 17. You may be wondering "Why?". And to keep it simple, I just didn't believe in myself. But as time passed I realized "Why do I have to wait until college to start doing the things that I like? It's not like the degree is going to write code."

It all started when I was a kid. I was always too busy in the computer playing videogames, back then, Flash Games. One day out of curiousity I asked "I wonder how do they make video games" to which my aunt replied what in my mind sounded like a bunch of non-sense, then all I heard in the end was "And then programming". That word alone sounded like magic. Which is what that sort of blind admiration for the topic.

10 years later and I was playing the mod for the game Grand Theft Auto: San Andreas, known as "San Andreas Multiplayer". I was very active in a server known as "Los Santos Roleplay" and also in the forums, which were made in phpBB (Nostalgia points). I saw that they had developers in the team and so I went ahead and Private Messaged the server owner "What do I have to learn to be able to program a San Andreas Multiplayer server?". To which he replied "Before getting into it I had previous knowledge in Java. However for SAMP what you need to learn is Pawn". And so that was my first contact with software development, learning Pawn to try to make a SAMP Server.

I went through many different languages starting with Pawn, learning Python in Codecademy which then turned into Java. Stubborn me however kept hearing the comment "C++ is very hard" so I picked it up cause I'm just cool like that. Stuck with it for the longest time until 2021 which is when I was introduced to JavaScript and by extension ended up going for Backend then Full Stack Web Development. And here we are now.
